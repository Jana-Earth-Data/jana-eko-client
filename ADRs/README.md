# Architecture Decision Records (ADRs)

This folder contains Architecture Decision Records for the jana-eko-client library.

## What is an ADR?

An Architecture Decision Record (ADR) is a document that captures an important architectural decision made along with its context and consequences.

## ADR Process

1. **Create a new ADR** using the template in `template.md`
2. **Number sequentially** (e.g., `0001-decision-title.md`)
3. **Status progression**: Proposed → Accepted → Superseded
4. **Never delete** an ADR - instead mark it as Superseded

## ADR Status Values

- **Proposed**: Decision is under discussion
- **Accepted**: Decision has been accepted and is in effect
- **Superseded**: Decision has been replaced by a newer ADR
